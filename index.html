变量提升

var命令会发生“变量提升”现象，即变量可以在声明之前使用，值为undefined。这种现象多多少少是有些奇怪的，按照一般的逻辑，变量应该在声明语句之后才可以使用。

看解释非常抽象，我们结合下面的例子来讲。

为什么需要块级作用域

ES5 只有全局作用域和函数作用域，没有块级作用域，这带来很多不合理的场景。

  1. 内层变量可能会覆盖外层变量。

    上面这个例子。执行f时，虽然定义了全局变量tmp,但是因为变量提升,f中定义的tmp，替代全局tmp。但是tmp的值却没有定义，导致f输出了undefined。

  2. 用来计数的循环变量泄露为全局变量。

    这个例子，变量i只用来控制循环，但是循环结束后，它并没有消失，泄露成了全局变量。

什么是let

了解了以上的基础，现在就可以说说let和const了。

  1. let和var都是定义变量的用的命令。但是let声明的变量只在块级作用域内有效。

  2. 不存在变量提升。ES6提出了let和const来防止变量提升可能导致的错误。它们所声明的变量一定要在声明后使用，否则报错。

  3. 暂时性死区。

    只要块级作用域内存在let命令，它所声明的变量就“绑定”（binding）这个区域，不再受外部的影响。

    上面这段代码，虽然外部已经声明了tmp,但是由于块内有let声明的tmp，在let完成声明之前，就是暂时性死区，既读不到var声明的tmp,也读不到let声明的tmp，所报错。

  1. 不允许重复声明。

什么是const

  1. const声明一个只读的常量。一旦声明，常量的值就不能改变。

  2. 因为const不能改变，所以声明时必须赋值。否则会报错。

  3. 块级作用域，暂时性死区，不会变量提升，不可重复声明的特性和let一致。

const的本质

const实际上保证的，并不是变量的值不得改动，而是变量指向的那个内存地址所保存的数据不得改动。对于简单类型的数据（数值、字符串、布尔值），值就保存在变量指向的那个内存地址，因此等同于常量。但对于复合类型的数据（主要是对象和数组），变量指向的内存地址，保存的只是一个指向实际数据的指针，const只能保证这个指针是固定的（即总是指向另一个固定的地址），至于它指向的数据结构是不是可变的，就完全不能控制了。因此，将一个对象声明为常量必须非常小心。

使用场景

解决for循环i变量泄露的问题

在为什么需要块级作用域中，提到了for循环中使用var定义i会导致i泄露的问题。使用let就可以避免出现泄露，保证循环功能正确。

let的出现优雅的解决了var变量提升导致循环绑定事件只能绑到最后一个的问题。
其他解决方案见for循环绑定事件解决方法

组件化框架与const

const定义Object对象时，可以保证指针不变，防止被重新赋值，可以保证框架内部组件的稳定性。如react框架。

前言

Hexo这个博客的形式没有线上编辑管理的功能。而且部署后，仓库里只有发布后的代码，其他地方无法管理，既不方便也不安全。这样明显很不cool。
所以需要学习一下如何多地管理。搜索了一下，其实就是再上传一份源码，部署归部署，源码归源码。

这里参考这篇文章——Hexo 实现多地部署及优化访问速度

同时再写一下自己的理解。

学习

先是去网上查了一下,结果发现大多数采用的方案都是在一个 git 仓库里开两个分支. 一个放静态网页, 一个放源码. 但是这样其实是很不安全的! 在 Hexo 中我们为了方便会加一些插件,而某些插件会用到第三方平台.比如笔者博客中的评论功能( Power by Valine ) 就用到了 LeanCloud 的服务. 将评论内容, 访问量等信息存储在容器里,而这个容器的 Token 是万万不能给别人的. 而网上的通用方案因为仓库公开, 所以任何人都能看到你在里面设置的 Token, 这样是不行的
